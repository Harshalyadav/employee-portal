Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Docker Verification

Build the production image locally with the same platform target used in CI:

```bash
npm run docker:build
```

If you need a clean rebuild with verbose logs:

```bash
npm run docker:build:no-cache
```

Run the built container locally:

```bash
npm run docker:run
```

Then verify these endpoints:

- `http://localhost:3000`
- `http://localhost:3000/health`

The Dockerfile explicitly installs Alpine musl-native bindings for Lightning CSS and Tailwind Oxide so the Next.js production build works consistently in Docker.
